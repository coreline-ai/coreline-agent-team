이 문서는 이번 개발의 범위를 고정하고, 구현이 목적 밖으로 확장되지 않도록 하기 위한 작업 문서다.

## 개발 목적

이번 개발의 목적은 `agent-team`을 단순 운영용 엔진/콘솔 수준에서 한 단계 올려, 사용자가 **자연어 goal 1개로 팀을 자동 부팅하고 실제 작업을 시작할 수 있는 최소 실사용 경로**를 여는 것이다.

특히 아래 목표를 가진다.

- `agent-team run "<goal>"` 형태의 사용자 진입점을 추가한다.
- goal을 기반으로 `planner`, `search`, `frontend`, `backend`, `reviewer` 서브 에이전트를 자동 구성한다.
- workspace, team, 초기 task, leader message를 자동 부팅한다.
- 사용자는 초기 수동 `init/spawn/task-create/send` 없이 바로 TUI/watch로 관찰 가능한 상태에 들어간다.

## 개발 범위

이번 문서에서 허용하는 범위는 아래에 엄격히 한정한다.

1. `run <goal>` CLI 진입점 추가
2. 기본 preset(`software-factory`) 기반 역할 자동 구성
3. workspace / team / bootstrap 문서 / 초기 task / 초기 mailbox message 자동 생성
4. background sub-agent 자동 launch
5. 역할 고정 task routing에 필요한 기존 런타임 정합성 보강
6. 테스트/README/examples/사용 방법 갱신

## 제외 범위

- direct OpenAI API 또는 기타 vendor direct API 연동
- 완전 자율 recursive team spawning 구현
- 동적 role composer / AI 기반 preset 자동 선택 전체 구현
- pane backend / remote transport / cross-machine 확장
- 새로운 TUI 기능 대형 추가
- 원본 제품 exact parity 확대

## 문서 기반 제약 사항

- 이 프로젝트의 LLM 사용 표준 경로는 `Codex CLI` runtime이다.
- direct API 연동은 범위 밖이며 금지한다.
- 이번 목표는 **최소 실사용자 경로**를 여는 것이지, 완전 자율 소프트웨어 공장 전체를 한 번에 완성하는 것이 아니다.
- 최초 vertical slice는 `software-factory` preset 고정으로 구현한다.
- 구현 중 발견되는 정합성 문제는 현재 구조 안에서 수정한다.

## 공통 진행 규칙

- 구현 중 발생한 이슈는 해당 Phase에서 수정하고 기록한다.
