## 개발 목적
이번 개발의 목적은 `atcli` / `agent-team app` 경로를 **실사용 직전 수준**으로 마무리하는 것이다.

이번 작업은 아래 두 가지만 우선한다.

- 사용자가 앱 안에서 현재 결과 상태와 생성된 파일을 바로 판단할 수 있게 한다.
- 실제 `Codex CLI` runtime으로 장시간 실사용 검증을 수행하고, 검증 중 드러난 blocker만 수정한다.

## 개발 범위
1. 앱 내부 result state를 실제 진행 상태(`running/completed/attention/pending`)로 표시
2. 생성 파일 목록과 핵심 결과물 preview를 앱 화면에 표시
3. 상태 변화/결과 감지 로그를 최소한으로 보강
4. 실제 `Codex CLI` runtime 장시간 검증 수행
5. 검증 중 발견된 blocker 수정
6. 필요한 최소 문서 갱신

## 제외 범위
- Codex CLI UI exact parity를 위한 cosmetic 입력 UX 리디자인
- 고급 slash command 확장
- preset/role/recursive spawning 고도화
- direct OpenAI API 또는 vendor API 연동
- 원격 협업/멀티머신 transport
