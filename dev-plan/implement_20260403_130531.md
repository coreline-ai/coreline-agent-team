## 개발 목적
이번 개발의 목적은 `agent-team`을 개발자용 엔진이 아니라 일반 사용자가 **설치하고, 실행 전에 점검하고, 바로 실행할 수 있는 실제 사용 환경**으로 정비하는 것이다.

특히 이번 작업은 아래를 우선한다.

- `agent-team doctor` 기반 실행 전 진단 경로 확보
- `npm link` 후 `agent-team ...` 형태의 실제 실행 흐름 정비
- `Codex CLI` 경로 / 로그인 / workspace 쓰기 가능 여부의 명확한 확인
- 후속 `attach`/결과 확인 UX를 위한 사용자용 baseline 출력 정리

## 개발 범위
1. 사용자 실행 전 환경 진단(`doctor`) 명령 추가
2. `Codex CLI` 경로 / 로그인 / 기본 exec probe 점검
3. workspace 경로 쓰기 가능 여부 점검
4. `npm link` 후 `agent-team` bin 직접 실행 가능 상태 보장
5. 관련 테스트와 사용자 문서 갱신

## 제외 범위
- direct OpenAI API 또는 기타 vendor direct API 연동
- 완전 자율 multi-agent 고도화 또는 recursive spawn 확대
- 대규모 TUI 개편
- 원본 제품 exact parity 확장
