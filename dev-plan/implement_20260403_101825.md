이 문서는 이번 개발의 범위를 고정하고, 구현이 목적 밖으로 확장되지 않도록 하기 위한 작업 문서다.

## 개발 목적

이번 개발의 목적은 `agent-team`의 남은 우선 과제 중 1순위인 **`Codex CLI` 중심 repeated live soak / burn-in 검증 자동화**를 구현해, 현재의 one-shot smoke 수준을 넘어 **반복 실행 신뢰도**를 지속적으로 검증할 수 있게 만드는 것이다.

특히 이번 작업은 아래 목표를 가진다.

- `Codex CLI` runtime 기준 반복 `spawn/resume/reopen/status/tasks/transcript` 흐름을 자동화한다.
- `--root-dir` 기반 격리 상태에서 soak 실행, 정리, 실패 재현을 반복 가능하게 만든다.
- 실패 시 task/member/transcript/session 상태를 운영자가 재현 가능한 형태로 수집한다.
- burn-in 중 발견되는 기존 구조의 정합성 문제는 이번 범위 안에서 즉시 수정한다.

## 개발 범위

이번 문서에서 허용하는 범위는 아래에 엄격히 한정한다.

1. `Codex CLI` 기준 repeated live soak / burn-in 시나리오 정의
2. 반복 soak 실행용 스크립트 또는 재현 가능한 CLI 절차 구현
3. soak 실패 시 상태 스냅샷/로그 수집 절차 추가
4. soak 중 드러나는 기존 정합성 문제 수정 및 회귀 테스트 보강
5. soak 결과를 반영한 운영 문서/README/AGENT 업데이트

## 제외 범위

- direct OpenAI API 또는 기타 vendor direct API 연동
- 신규 runtime/backend 추가
- upstream `claude` CLI 중심의 별도 대형 자동화 구축
- TUI 재설계 또는 새로운 UI 기능 추가
- pane backend / cross-machine / remote transport 확장
- 원본 제품 exact parity 확대를 위한 별도 대형 개발
- 현 범위와 무관한 구조 개편

## 문서 기반 제약 사항

- direct API 연동은 범위 밖이며 금지한다.

## Phase 0. soak 기준선과 성공 기준 고정

### 목표
- `Codex CLI` 기준 repeated soak 의 명령 순서, 반복 횟수, 성공/실패 판정 기준을 고정한다.

### 구현 태스크
- [x] soak 대상 명령 흐름을 `init -> task-create -> spawn -> status -> tasks -> transcript -> resume -> reopen -> cleanup` 기준으로 고정한다.
- [x] `--root-dir` 디렉터리 규칙, 임시 경로 생성 규칙, 정리 규칙을 문서화한다.
- [x] 최소 반복 횟수와 통과 조건을 정량화한다. 예: 전체 반복 5회 이상, 실패 0건, orphan task 0건.
- [x] soak 중 확인해야 할 관찰 포인트를 정리한다. (`inactive worker`, `open task`, `session`, `transcript`, `permission drift`)
- [x] 실패 시 수집할 상태 스냅샷 항목을 정의한다. (`status`, `tasks`, `transcript`, 관련 저장 파일 경로)

### 자체 테스트
- [x] 기준선 CLI 명령 세트가 실제 현재 구현과 일치하는지 확인한다.
- [x] 1회 smoke로 기준선 절차를 실제 실행해 문서 초안과 결과가 맞는지 확인한다.
- [x] `resume`은 새 session, `reopen`은 기존 session 재사용이라는 기대 결과를 실제로 검증한다.
- [x] 반복 횟수/성공 기준/실패 판정 문장이 모호하지 않게 정의되었는지 확인한다.

### 이슈 및 수정
- [x] CLI exact 출력 형식이 문서의 `session=new-session` 표현과 달라 `(new-session)` / `(existing-session)` 기준으로 정리했다.
