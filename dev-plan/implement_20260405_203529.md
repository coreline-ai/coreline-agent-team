## Phase 1. shared summary / preview policy 정리

### 목표
- large-output workspace에서도 summary와 preview 선택 기준이 흔들리지 않게 공통 정책을 먼저 고정한다.
- `attach`와 `project builder`가 서로 다른 파일 수 제한/preview 절단 규칙을 쓰는 상태를 줄인다.

### 구현 태스크
- [x] `src/team-cli/commands/summary-utils.ts`의 summary / preview 계약을 점검하고, large-output 상태를 설명할 수 있는 공통 metadata(예: hidden count, truncated notice, preview selection 결과)를 정리한다.
- [x] preview 우선순위 heuristic을 보강해 대량 파일 세트에서도 low-signal 후보보다 핵심 산출물 문서/엔트리 파일을 더 안정적으로 고른다.
- [x] 파일 목록 표시 limit, preview line/byte/length 절단 규칙을 공통 helper 기준으로 정리한다.

### 자체 테스트
- [x] `tests/team-cli/summary-utils.test.ts`에 large-output summary / preview candidate / truncation 회귀 테스트를 추가한다.
- [x] `npm run typecheck`
- [x] `npm test` — 총 186 tests 통과

### 이슈 및 수정
- [x] 현재 `attach`와 `project builder`가 서로 다른 workspace file limit을 사용하고 있어, 공통 정책 정리 시 surface별 차이를 의도적으로 유지할지 함께 판단한다.
- [x] `project builder`가 `listWorkspaceFiles(..., 12)`를 따로 쓰던 경로를 공통 snapshot helper 기준으로 맞춰서, 이후 Phase 2에서 같은 metadata를 재사용할 수 있게 정리했다.
- [x] preview excerpt가 heading만 반복하던 케이스를 줄이기 위해, 본문 첫 줄이 있으면 excerpt가 heading 대신 본문을 사용하도록 정리했다.

## Phase 2. attach / project builder large-output UX polish

### 목표
- generated files가 많은 팀에서도 사용자가 핵심 산출물과 preview를 더 빨리 찾게 만든다.
- `attach`와 `project builder app`의 메시지/패널 표현을 가능한 한 같은 어휘로 맞춘다.

### 구현 태스크
- [x] `src/team-cli/commands/attach.ts`의 `generated files:` / `preview:` 섹션을 large-output friendly하게 정리한다.
- [x] `src/team-tui/project-builder-app.tsx`의 `Files` / `Preview` detail pane이 Phase 1의 공통 summary/preview metadata를 사용하도록 맞춘다.
- [x] hidden files, trimmed preview, selected preview path 같은 핵심 상태가 attach와 project builder에서 같은 의미로 보이도록 정리한다.

### 자체 테스트
- [x] `tests/team-cli/attach-command.test.ts`에 large-output generated files / preview message 회귀 테스트를 추가한다.
- [x] `tests/team-tui/project-builder-app.test.tsx`에 large-output preview / hidden files / trimmed notice 회귀 테스트를 추가한다.
- [x] `npm run typecheck`
- [x] `npm test` — 총 188 tests 통과

### 이슈 및 수정
