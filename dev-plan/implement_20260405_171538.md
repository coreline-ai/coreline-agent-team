| 빌드 품질 | 95% | 98% | outdated deps 경고, @types/react 위치, engines 미설정 |
| 코드 설계 | 85% | 95% | 폴링 루프 3곳 복붙(-7%), 로깅 프레임워크 부재(-5%), run.ts 비대(-3%) |
| 런타임 안정성 | 80% | 95% | 실패 시나리오 테스트 0건(-10%), 로깅 없음(-5%), stale lock 의존(-5%) |

직전 안정성 수정 작업에서 이미 완료된 항목 (이 계획의 선행 작업):

- ✅ Codex CLI / Upstream CLI 프로세스 타임아웃 (5분 기본, SIGTERM → SIGKILL)
- ✅ stdout/stderr 버퍼 50MB 제한
- ✅ 3개 폴링 루프 에러 핸들링 + 재시도 제한 (MAX_POLL_ERRORS=20, exponential backoff)
- ✅ Task claim 실패 시 롤백
- ✅ SIGTERM/SIGINT 핸들러 (bin.ts)
- ✅ Temp 디렉토리 finally 정리
- ✅ FD 누수 방지 (background-process.ts)
- ✅ JSON readJsonFile retry 범위 확대 + exponential backoff
