## 목적

`agent-team-module`의 Codex runtime에서 재현된 frontend stuck 증상 중,
이번 workstream에서는 아래 두 가지를 실제 코드로 개선한다.

1. active turn 도중 shutdown request가 들어와도 worker가 장시간 응답하지 않는 문제
2. timeout/interrupt 시 SIGTERM 이후 프로세스가 남아도 강제 종료로 수렴하지 않는 문제

## 범위

- `src/team-runtime/runtime-adapter.ts`
- `src/team-runtime/codex-cli-bridge.ts`
- 관련 타입/테스트 파일

## 제외 범위

- partial transcript flush의 완전한 구현
- task status reconciliation 모델 확장 (`partial`, `timed_out` 등)
- frontend role 전용 timeout 튜닝 surface 추가
- operator/dashboard 표현 변경

## 핵심 가설

- 현재 shutdown command는 mailbox에 `shutdown_request`만 쓰고, active turn 중인 worker는 이를 즉시 소비하지 못한다.
- 현재 Codex bridge는 SIGTERM 이후 실제 프로세스 종료를 기다리기보다 exit code를 먼저 반환하고, `child.killed`를 잘못 사용해 SIGKILL escalation이 약하다.
- 따라서 active turn을 중단시키는 per-turn abort signal과, 실제 close까지 수렴하는 강제 종료 경로가 필요하다.
