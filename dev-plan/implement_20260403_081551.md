## 이번 개발의 목적

이번 개발의 목적은 `agent-team`의 **실행 안정성, 상태 정합성, 유지보수성**을 개선하는 것이다.

특히 아래 문제를 우선 해결한다.

- worker 종료 후 task가 `in_progress`로 남는 상태 정합성 문제
- mailbox message의 처리/ack 시점 문제
- 파일 저장의 crash-safe 보강 필요
- 비대해진 CLI 진입점과 중복 로직 정리
- runtime / operator / TUI의 의미 중복 및 관리 비용 감소

또한 이번 개발은 아래 전제를 절대 흔들지 않는다.

- **LLM 사용의 표준 경로는 `Codex CLI` runtime이다.**
- direct OpenAI API 또는 기타 vendor API 연동은 **고려 대상이 아니며 금지**한다.
- 이번 안정화/리팩터링은 모두 **CLI 기반 agent runtime 유지**를 전제로 한다.

## 이번 개발의 범위

이번 문서에서 허용하는 범위는 아래에 **엄격히 한정**한다.

1. 기존 기능의 **정합성 수정**
2. 기존 구조의 **리팩터링**
3. 기존 동작을 보호하기 위한 **자체 테스트 추가/수정**
4. 문서와 구현의 **드리프트 수정**

## 강력한 범위 제한

아래 작업은 **이번 개발 범위에서 금지**한다.

- 새로운 제품 기능 추가
- 새로운 UI/UX 방향 실험
- 새로운 runtime/backend 추가
- remote transport, pane backend, cross-machine 기능 확장
- direct upstream parity 확대를 위한 별도 대형 개발
- OpenAI/기타 vendor direct API 연동
- 현 범위와 무관한 설계 변경

즉, 이번 개발은 **안정화 + 리팩터링 + 테스트 보강**만 수행한다.
